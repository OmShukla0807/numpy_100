# How I Explore Constructors and Class Documentation

When I encounter a new class such as `np.datetime64`, my goal is **not** to memorize the API. My goal is to discover:

1. What problem does this class solve?
2. How can I create instances?
3. What operations are supported?
4. What external concepts or standards does it depend on?

---

## Step 1: Read the First Few Lines

```python
help(np.datetime64)
```

Look for:

* Purpose of the class
* Accepted input formats
* Examples

Example:

```text
If created from string, the string can be in ISO 8601 date
or datetime format.
```

Important observation:

> "ISO 8601" is an external standard.

At this point I can search:

```text
ISO 8601 datetime examples
```

instead of guessing valid formats.

---

## Step 2: Read the Examples

Examples are often more valuable than the formal description.

```python
np.datetime64('1980')
np.datetime64(10, 'D')
```

From these examples I can infer constructor signatures.

Possible constructor patterns:

```python
np.datetime64(string)
np.datetime64(integer, unit)
```

---

## Step 3: Ignore Most Magic Methods Initially

Documentation may contain:

```text
__eq__
__lt__
__gt__
__add__
__sub__
```

Translate them into normal Python syntax:

```python
a == b
a < b
a > b
a + b
a - b
```

This tells me what operations the object supports.

---

## Step 4: Create an Object Immediately

Never stay in documentation too long.

```python
d = np.datetime64('2026-06-03')
```

Then inspect:

```python
type(d)
d.dtype
dir(d)
```

---

## Step 5: Explore Public Attributes

```python
[x for x in dir(d) if not x.startswith('_')]
```

Then try interesting names:

```python
d.astype(...)
d.itemsize
d.dtype
```

---

## Step 6: Experiment Aggressively

Documentation often gives hints, not complete answers.

Try:

```python
np.datetime64('2026')
np.datetime64('2026-06')
np.datetime64('2026-06-03')
np.datetime64('2026-06-03T10:30')
```

Observe what works.

---

## Step 7: Recognize External Standards

When documentation mentions terms like:

* ISO 8601
* UTF-8
* IEEE 754
* RFC 3339
* POSIX

assume they are established standards.

Search them directly instead of guessing.

---

## Constructor Exploration Checklist

When I discover a new class:

```python
help(MyClass)
```

1. Read purpose.
2. Read examples.
3. Identify constructor patterns.
4. Create an instance.
5. Inspect with `dir()`.
6. Experiment with inputs.
7. Search unfamiliar standards or terminology.

The fastest way to learn a class is:

Documentation → Example → Experiment.
